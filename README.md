# 出発準備45分！ Airport Turnaround

**▶ 遊ぶ: https://esma-dev-studio.github.io/airport-turnaround/**

飛行機が空港に到着してから、次の便として出発するまでの地上作業(ターンアラウンド)を管理する
ブラウザ用シミュレーションゲームです。プレイヤーは限られたスタッフ・車両・時間をやりくりし、
安全を守りながら定刻出発を目指します。

- 対象: 小学校高学年〜大人
- プレイ時間: 1ステージ 5〜10分(ゲーム内45分 ＝ 1倍速で実時間6分)
- 舞台: 架空の「そらみなと国際空港」/架空の「あおぞら航空」

## 遊び方

1. `index.html` をブラウザで開く(そのままダブルクリックで動作します)
2. タイトル → ステージ選択 → ステージ1(クリアすると次のステージが解放されます)
3. 作業のすすめ方は2とおり:
   - **おまかせ**: 左の📋ドロワーの作業カードで「開始」→ 必要なメンバーが自動で移動
   - **自分で配置**: 駐機場の待機所にいる**スタッフ・車両をタップ**して選び、**緑に光る持ち場をタップ**して送りこむ(必要数がそろうと作業開始)
4. 各作業は **🐢ていねい/ふつう/🚀急ぐ** の采配レバーつき(急ぐと速いが満足度・安全性が下がる)。スタッフには **⭐ベテラン(15%速い)/🔰新人(15%ゆっくり)** がいて、だれを送るかも作戦のうち
5. 駐機場はドラッグで移動、ホイール/ピンチ/右下の＋−でズーム
6. ⏸/▶/⏩ で時間の速さを切り替え(スペースキー＝一時停止、1/2キー＝速度)
7. 12の作業をすべて終えて、安全に出発できればクリア。100点満点で評価されます

## ステージ

| ステージ | 難易度 | 内容 |
|---|---|---|
| 1 はじめての出発準備 | 初級 | 晴天・トラブル最小限。常時ヒントつきで基本操作を学ぶ。イベント1件(遅れている乗客) |
| 2 定刻出発を目指せ | 中級 | ゲート係1人・ランプ係1人・カート1台と人手不足。清掃スタッフ遅延+点検で異常発見の2イベント。ヒントは序盤のみ |
| 3 消えた手荷物 | 中級 | 雷雨で屋外作業が一時停止、手荷物1個が行方不明、さらに遅れる乗客。3イベントの総合戦。ヒントは序盤のみ |

## 実装した機能

- **全画面の駐機場ビュー**: 駐機場が画面いっぱいに広がり、作業カードは左の折りたたみドロワー、指標は右上、スタッフ・車両は下のバーとしてオーバーレイ表示。カメラはズーム(ホイール/ピンチ/ボタン)とドラッグ移動に対応
- **タップで直接配置**: 待機所のスタッフ・車両をタップ→受け入れ可能な持ち場が緑に光る→タップで送りこみ。1人ずつの部分割当と「おまかせ」を併用できる。持ち場の点線サークルをタップするとその場でミニメニュー(開始・中断・ペース変更)
- **采配レバー**: 対象9作業に「ていねい(×1.2・満足度+)/ふつう/急ぐ(×0.8・満足度−、給油・点検・照合は安全性−)」
- **ゲーム画面レイアウト**: 上部HUD(時刻・出発予定・残り時間・速度切替)+上記オーバーレイ構成
- **ゲーム内時間**: 一時停止・通常・2倍速(ゲーム内1分＝実時間8秒)。残り時間はマイナス表示せず「+X分遅れ」表示
- **12の作業カード**: 降機→取り降ろし→清掃→機内食→給油→点検→積み込み→搭乗→照合→ドアクローズ→プッシュバック→出発。状態は「未開始(ロック)/移動中/作業中/停止中/問題発生/完了」を色分け表示
- **作業間の依存関係**: 清掃は降機後、搭乗は清掃後、照合は積み込み+搭乗後、ドアクローズは搭乗・照合・機内食・給油後、プッシュバックはドアクローズ+点検後。条件未達で開始しようとすると理由を表示
- **並行作業**: 降機と同時に取り降ろし・給油・点検を進められるなど、実際の空港と同様の同時作業が可能
- **リソース管理**: スタッフ5種・車両5種を個別管理。二重割り当て不可、不足時は理由表示、「中断」で呼び戻し(進捗保存)
- **駐機場ビュー(Canvas 2.5D風)**: 機体・搭乗橋の伸縮・形の違う5車両・スタッフ移動・乗客の流れと人数カウンター・給油ホースと安全コーン・点検の機体一周・雷雨の雨エフェクト・作業完了の紙ふぶき演出・プッシュバック〜出発アニメ
- **突発イベント5種をすべて実装**:
  1. 手荷物が1個見つからない(ステージ3・積み込み中に発生)
  2. 清掃スタッフの到着遅延(ステージ2)
  3. 機体点検で軽微な異常を発見(ステージ2・点検の進捗60%で発生)
  4. 搭乗に遅れている乗客(ステージ1・3)
  5. 天候悪化=雷雨(ステージ3・屋外作業が一時停止。屋内作業は継続できる)
  - ゲームを止めない通知バナー→選択式モーダル(2〜3択、正解は1つではない)。放置すると期限で自動対応。対応中の作業カードは「⚠問題発生」表示
- **安全確認の選択**: プッシュバック前に「確認(+1分)/省略」。省略や、点検異常の確認省略・雷雨中の作業続行は安全性が大きく減点され、出発してもクリア扱いになりません
- **スコア**: 安全性40+定時性30+乗客満足度20+コスト効率10。ランクS〜D(安全に重大な問題があれば「×」)。結果画面に内訳・出発時刻・文章での改善アドバイス(開始が遅かった作業やイベント対応へのひとことを自動生成)・**作業タイムライン(ガントチャートで並行度を可視化)**・空港まめ知識
- **プレイ中のフィードバック**: 指標が動いた瞬間の「±N」表示、遅れ気味のときの残り15/10/5分警告、一時停止中の表示、ドロワーの「開始できる作業数」バッジ
- **学べる要素(子ども向けの理解サポート)**:
  - 🗺 **出発準備のながれマップ**: 12作業の順番と「同時にできる作業」をフローチャートで表示。ゲーム中はHUDの「🗺ながれ」から開け、完了/作業中/開始できる/まだ が色でわかる(開くと自動で一時停止)
  - 📖 **空港ずかん**(タイトル画面): ながれ/おしごと12種(それぞれに「💡なぜ必要?」)/スタッフと車両の3タブ
  - ✈ **到着シネマティック**: ステージ開始時に飛行機が誘導路から入ってきて駐機→搭乗橋が伸びて作業開始。「到着から出発まで」の物語が一巡見える(タップでスキップ可)
  - 👨‍✈️ **コーチのふきだし**(ステージ1): 「お客さんが降りたら清掃だよ」など、流れの節目を実況ガイド
  - チュートリアルにも「ながれ」ページを追加
- **ステージ進行**: クリアで次ステージ解放。結果画面に「▶次のステージへ」ボタン
- **ヒント**: 初級は常時「次のおすすめ」を表示、中級は序盤8分のみ(ヒントを限定)
- **チュートリアル**: 初回起動時に5ページで表示。設定からいつでも再表示可
- **効果音**: WebAudioで生成(外部ファイルなし)。設定でオン/オフ、保存される
- **セーブ**: クリア済みステージ・ベストスコアを localStorage に保存(再読み込みでも保持)
- **レスポンシブ**: PC 3カラム/タブレット 2カラム/スマホ縦1カラム
- **時間切れ処理**: 出発予定+30分(中級は+25分)で強制終了(未出発扱い)

## 既知の制約

- ブラウザのタブが非表示のときはゲームが進みません(見ていない間に失敗しない仕様でもあります)
- 1フレームの経過時間は最大0.25秒に制限しているため、極端に重い環境では実時間よりゆっくり進むことがあります
- スタッフの人形は視認性のため実際の縮尺よりやや大きめに描いています(機体と車両の縮尺は統一)
- 効果音はブラウザの自動再生制限のため、最初のクリック以降に有効になります
- イベントの発生順は1件ずつ(同時に複数のイベントは発生しません)

## ファイル構成

| ファイル | 責務 |
|---|---|
| `index.html` | 画面構造(タイトル/選択/ゲーム/結果/モーダル) |
| `styles.css` | デザインシステム・レスポンシブ |
| `main.js` | 起動・画面遷移・ゲームループ・速度制御・セーブ・ステージ解放 |
| `game.js` | ゲームエンジン(時間・作業状態・リソース割当・イベント・指標・スコア)※DOM非依存 |
| `tasks.js` | 12作業とスタッフ・車両の定義データ(屋外/屋内の区分も) |
| `events.js` | 突発イベント5種の定義データ |
| `stages.js` | ステージ設定(リソース数・イベント・時刻・ヒント方針) |
| `ui.js` | DOM描画・モーダル・効果音・駐機場Canvasビュー |
| `assets/` | タイトル画像・ファビコン(自作SVG) |

ゲームデータ(作業・イベント・ステージ)はすべてJavaScriptオブジェクトで定義しており、
ファイルを1つ編集するだけで追加・調整できます。イベントは発生時刻(`at`)だけでなく
作業の進捗率(`when: {task, pct}`)でも発生させられます。
